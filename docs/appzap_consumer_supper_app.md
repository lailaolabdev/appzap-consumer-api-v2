**Vision:** The "Smart Lifestyle OS" for Laos. One app to Eat, Live, and Buy.

## **1. CORE NAVIGATION ARCHITECTURE (The "Trinity" Layout)**

The app uses a **Hybrid Navigation Model** to separate the 3 products cleanly.

### **1.1 Global Navigation Elements**

  * **Top Header (The Vibe Toggle):** A segmented control switch at the very top.
      * **Left:** `[ EATS 🔴 ]` (Default/Dark Mode).
      * **Right:** `[ LIVE 🟢 ]` (Health/Light Mode).
      * *Behavior:* Tapping switches the entire UI theme and Home Feed content.
  * **Bottom Navigation Bar (The Utility Anchor):** Always visible.
      * Tab 1: **Home** (Dynamic based on Top Toggle).
      * Tab 2: **Market** (The Supply Store - Product 3).
      * Tab 3: **Activity** (Orders/Bookings).
      * Tab 4: **Account** (Profile/Wallet).

## **2. AUTHENTICATION & PROFILE SYSTEM (The "Smart Switch")**

**Requirement:** Handle Household (B2C) and Restaurant (B2B) users seamlessly.

### **2.1 Mobile OTP Login**

  * **Flow:** User enters Phone Number $\rightarrow$ Request OTP $\rightarrow$ Verify.
  * **Database Logic (MongoDB `Users` Collection):**
      * `phone`: +85620... (Unique Key).
      * `roles`: `['consumer']` OR `['consumer', 'merchant_owner']`.
      * `linked_restaurants`: `[Restaurant_ID_1, Restaurant_ID_2]`.

### **2.2 The "Profile Switcher" (Market Tab Only)**

  * **Location:** Top Right corner of the **Market Tab**.
  * **UI:** Avatar Icon with a small "Repeat" arrow (Like Facebook Page Switcher).
  * **Logic:**
      * **Default:** "Personal Profile" (B2C Interface).
      * **Action:** Tap Icon $\rightarrow$ Slide-up Sheet showing available profiles:
        1.  **Somsack (Personal)**
        2.  **Noy’s BBQ Shop (Merchant)**
      * **Result:** Switching to "Merchant" changes the Market UI to **Wholesale Mode** (Bulk prices, Credit Terms).

## **3. PRODUCT 1: APPZAP EATS 🔴 (Consumption Engine)**

**Vibe:** Fun, Social, Fast.
**MVP Priority:** High (Phase 1 Features).

### **"AppZap Rich Discovery."** Don't make them scroll endlessly. Help them Find the Perfect Spot instantly.

Here is the Revised Phase 1 Feature that replaces the Video Feed.

1. Feature: "AppZap Genie" (The AI Search)
The Logic: Use MongoDB Vector Search (which is built-in and cheap).

How it works:

You don't need to train a complex AI.

You store restaurant "Tags" and "Reviews" as vectors.

User Experience:

User types: "Romantic dinner near the river with live folk music."

AppZap shows: "Kong View Restaurant" (98% Match).

Why it wins: Foodpanda cannot do this. They only search "Sushi" or "Burger." You search Vibe.

2. Feature: "Living Profiles" (Video Headers)
The Logic: Don't build a Feed. Build Video Headers.

Implementation:

When a user clicks a Restaurant Profile, the Header Image is a Video (Auto-play, muted).

It shows the atmosphere (sizzling steak, jazz band, pouring beer).

Why it wins: It gives the "Vibe Check" feeling without the complexity of a TikTok algorithm. It makes the menu feel "Alive."

### **3.2 "Cash Flow" Ordering Engine (P0 - Priority)**

  * **Smart Order Toggle:** Switch between `[Dine-In]` and `[Takeaway]` at the top of the menu.
  * **One-Click Upsell:** Before payment, show "Add Cheesy Fries" modal with 10s timer.
  * **Voucher Marketplace:** Buy 100k credit for 75k. (Scarcity countdown).
  * **Tip the Kitchen:** Post-payment micro-transaction.

### **3.3 Live Bill & Split (Dine-In)**

  * **Deep Link Trigger:** Scanning table QR opens specific bill session.
  * **Live Sync:** Socket.io connection to POS updates bill in real-time.
  * **Split Logic:** "Split 4 ways" $\rightarrow$ Generates 4 payment links.

## **4. PRODUCT 2: APPZAP LIVE 🟢 (Longevity Engine)**

  * **Flow:** Select Goal (Muscle/Weight Loss) $\rightarrow$ Pay Upfront ($110) $\rightarrow$ Auto-order creation daily.

### **4.3 Supplement Upsell (The "Power Up")**

  * **Location:** Checkout Screen (Cross-product).
  * **UI:** Carousel above "Total".
  * **Items:** Collagen, Vitamin C.

## **5. PRODUCT 3: APPZAP MARKET 🔵 (Supply Engine)**

**Vibe:** Blue, Logistic, Efficiency.
**Requirement:** Dynamic UI based on Profile.

### **5.1 View A: Household (B2C)**

  * **Target:** The user cooking dinner.
  * **UI Layout:** "Supermarket Style" (Grid view).
  * **Inventory:** Small packs (e.g., 500g Pork, 6-pack Beer).
  * **Pricing:** Retail Price.
  * **Checkout:** Immediate Payment (Card/QR).

### **5.2 View B: Restaurant (B2B)**

  * **Target:** The owner restocking for tomorrow.
  * **Trigger:** Profile Switched to "Merchant".
  * **UI Layout:** "Order Sheet Style" (List view, quick quantity entry).
  * **Inventory:** Bulk packs (e.g., 50kg Pork Sack, 20 Crates Beer).
  * **Pricing:** Wholesale Price (Tiered pricing).
  * **Features:**
      * **"Re-order Last Stock":** One-click duplicate of last week's order.
      * **Credit Payment:** "Pay Later" option (if approved).
      * **Tax Invoice:** Checkbox to email PDF invoice.

## **6. CRITICAL SHARED MODULES (Technical Specs)**

### **6.1 The "Web-to-App" Bridge (Deep Linking)**

  * **Tech:** Firebase Dynamic Links / Branch.io.
  * **Scenario:**
    1.  User on Web Menu adds items to cart.
    2.  User taps **"Download App for Discount"**.
    3.  **App installs.**
    4.  **On First Launch:** App grabs `deep_link_data` containing `{ cart_items: [...], table_id: 123 }`.
    5.  **Action:** App navigates immediately to `CheckoutScreen` with data pre-filled.

### **6.2 Loyalty Engine (Points)**

  * **Logic:**
      * `Earn`: `Order_Total / 1000`.
      * `Burn`: Rate of 10 pts = 500 LAK.
  * **UI:** Wallet Balance on Home Screen. "Slide to Redeem" on Checkout.
  * **Welcome Hook:** First-time login triggers 15-minute countdown for Free Beer.

### **6.3 Payment Gateway**

  * **SDK:** Phapay Integration.
  * **Handling:**
      * **App Jump:** App $\rightarrow$ BCEL One $\rightarrow$ App.
      * **Callback:** Webhook updates Order Status in MongoDB to `PAID`.
      * **POS Sync:** If `PAID`, send print command to Kitchen Printer immediately via Socket.

## **7. DEVELOPER USER FLOWS (Happy Paths)**

### **Flow A: The "Trap" (Web to App)**

1.  **Web:** User scans QR $\rightarrow$ Adds Burger $\rightarrow$ Clicks "Download & Save".
2.  **Store:** User installs AppZap.
3.  **App:** Splash Screen $\rightarrow$ Phone OTP Login.
4.  **Magic:** App detects `Table 123` context $\rightarrow$ Opens Checkout $\rightarrow$ Shows Burger $\rightarrow$ Applies Discount.

### **Flow B: The "Party to Health" Switch**

1.  **Home:** User is in **EATS** mode (Red). Scrolls Restaurant List.
2.  **Action:** Toggles Top Switch to **LIVE** (Green).
3.  **UI Update:** Theme turns White/Green. Feed changes to "Healthy Bowls" and "Fasting Timer".
4.  **Action:** Clicks "Order Salad".
5.  **Checkout:** Adds "Collagen Shot" from Upsell Carousel. Pays.

### **Flow C: The "Merchant Restock"**

1.  **Tab:** User clicks **MARKET** tab.
2.  **Switch:** Taps Profile Icon $\rightarrow$ Selects "Noy's BBQ Shop".
3.  **UI Update:** Prices drop to Wholesale. Items change to Bulk.
4.  **Action:** Clicks "Re-order Last Week".
5.  **Checkout:** Selects "Credit Term (30 Days)" payment. Confirms.
