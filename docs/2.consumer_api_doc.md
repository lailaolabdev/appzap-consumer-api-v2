## Overview

### Purpose

The **AppZap Consumer API** is a unified gateway API that orchestrates the "AppZap Super App" mobile experience. It consolidates three distinct products into one seamless consumer journey:

- **🔴 AppZap Eats**: Restaurant discovery, ordering, and dining
- **🟢 AppZap Live**: Health-focused meals and wellness subscriptions
- **🔵 AppZap Market**: B2C/B2B grocery and ingredient shopping

### Design Philosophy

**"One User, Multiple Contexts"**

The API enables a single authenticated user to:
- Switch between consumption modes (Eats vs Live)
- Toggle between profile types (Personal vs Merchant)
- Access different inventory systems (Restaurant POS vs Supplier Warehouse)

### Base URLs

- **Production**: `https://consumer-api.appzap.la`
- **Staging**: `https://staging-consumer-api.appzap.la`
- **Development**: `http://localhost:9000`

### Technology Stack

- **Runtime**: Node.js 18+ (Express.js)
- **Database**: MongoDB 6.0+
- **Cache**: Redis (for tokens, sessions)
- **Real-time**: Socket.io (for live bills, order tracking)
- **Payment**: Phapay SDK Integration
- **Deep Linking**: Firebase Dynamic Links

## Endpoint Implementation Map

This section clarifies **WHERE each endpoint is implemented** and **WHAT data source it uses**.

### Implementation Types

| Symbol | Type | Description |
|--------|------|-------------|
| 🟢 **Direct** | Consumer API implements the logic and queries Consumer DB | New code in Consumer API |
| 🔵 **Proxy** | Consumer API forwards request to external API | Passthrough with minimal processing |
| 🟠 **Orchestration** | Consumer API combines data from multiple sources | Aggregates data from multiple APIs |
| 🟣 **Hybrid** | Consumer API adds logic + proxies to external API | Enriches external data with local data |

## Authentication

### Flow Overview

The Consumer API uses **OTP-only authentication** with no passwords.

### 1. Request OTP

**Endpoint**: `POST /v1/auth/request-otp`

**🔵 Implementation Type**: **Proxy**  
**📡 Data Source**: Auth API (GraphQL)  
**🔗 External Endpoint**: `https://auth.lailaolab.com` → `requestOtp` mutation

**Description**: Initiates OTP request via Auth API (GraphQL proxy)

**Request Body**:
```json
{
  "phone": "8562093352677",
  "platform": "APPZAP",
  "header": "AppZap"
}
```

**Response** (200 OK):
```json
{
  "message": "OTP sent successfully",
  "referenceId": "OTP-2025-001",
  "expiresIn": 300
}
```

**Implementation Logic**:
```javascript
// Consumer API Controller
async requestOTP(req, res) {
  const { phone, platform, header } = req.body;
  
  // 1. Check rate limit (Redis)
  const rateLimitKey = `otp:rate:${phone}`;
  const attempts = await redis.incr(rateLimitKey);
  if (attempts > 3) {
    return res.status(429).json({ error: 'Too many requests' });
  }
  await redis.expire(rateLimitKey, 300); // 5 minutes
  
  // 2. Proxy to Auth API (GraphQL)
  const result = await authApiClient.mutation({
    mutation: REQUEST_OTP_MUTATION,
    variables: { data: { phone, platform, header } }
  });
  
  // 3. Return response
  return res.json(result.data.requestOtp);
}
```

**Configuration Required**:
- Environment variable: `AUTH_API_URL=https://auth.lailaolab.com`
- Redis for rate limiting

---

### 2. Verify OTP & Login

**Endpoint**: `POST /v1/auth/verify-otp`

**🟠 Implementation Type**: **Orchestration**  
**📡 Data Sources**: 
1. Auth API (GraphQL) - OTP verification
2. Consumer DB (MongoDB) - User management

**🔗 External Endpoint**: `https://auth.lailaolab.com` → `phoneLogin` mutation

**Description**: Verifies OTP and creates/retrieves user session

**Request Body**:
```json
{
  "phone": "8562093352677",
  "otp": "123456"
}
```

**Response** (200 OK):
```json
{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": "user_123",
    "phone": "8562093352677",
    "fullName": "Somsack Phonevilay",
    "roles": ["consumer"],
    "profiles": [
      {
        "type": "personal",
        "name": "Somsack (Personal)",
        "isActive": true
      }
    ],
    "hasCompletedOnboarding": false,
    "firstLogin": true,
    "createdAt": "2025-12-18T10:00:00.000Z"
  }
}
```

**Implementation Logic**:
```javascript
// Consumer API Controller
async verifyOTP(req, res) {
  const { phone, otp } = req.body;
  
  // 1. Verify OTP with Auth API
  const authResult = await authApiClient.mutation({
    mutation: PHONE_LOGIN_MUTATION,
    variables: { where: { phone, otp } }
  });
  
  if (!authResult.data.phoneLogin) {
    return res.status(400).json({ error: 'Invalid OTP' });
  }
  
  // 2. Check if user exists in Consumer DB
  let user = await User.findOne({ phone });
  
  // 3. Create user if not exists
  if (!user) {
    user = await User.create({
      phone,
      authProviderId: authResult.data.phoneLogin.data.id,
      roles: ['consumer'],
      activeProfile: 'personal',
      points: { balance: 0, tier: 'bronze' },
      hasCompletedOnboarding: false,
      firstLogin: true
    });
    
    // Initialize loyalty points
    await LoyaltyTransaction.create({
      userId: user._id,
      type: 'bonus',
      amount: 0,
      description: 'Account created'
    });
  } else {
    // Update existing user
    user.lastLogin = new Date();
    user.firstLogin = false;
    await user.save();
  }
  
  // 4. Generate Consumer API JWT tokens
  const accessToken = jwt.sign(
    { userId: user._id, phone: user.phone },
    process.env.JWT_SECRET,
    { expiresIn: '24h' }
  );
  
  const refreshToken = jwt.sign(
    { userId: user._id },
    process.env.JWT_REFRESH_SECRET,
    { expiresIn: '30d' }
  );
  
  // 5. Return user with tokens
  return res.json({
    accessToken,
    refreshToken,
    user: user.toJSON()
  });
}
```

**Database Operations**:
- **READ**: Query `Users` collection by `phone`
- **WRITE**: Insert new user OR update `lastLogin`
- **WRITE**: Insert loyalty transaction record

**Error Responses**:
- `400` - Invalid OTP
- `429` - Too many attempts
- `401` - OTP expired

### 1.1 Restaurant Discovery

#### Get Restaurants (with Mode Toggle)

**Endpoint**: `GET /v1/eats/restaurants`

**🟣 Implementation Type**: **Hybrid**  
**📡 Data Sources**: 
1. POS V2 API - Restaurant data (name, location, menu)
2. Consumer DB - Health tags, ratings, reviews

**🔗 External Endpoint**: POS V2 API → `GET /api/stores` or similar

**Description**: Returns restaurants based on current mode (Eats vs Live)

**Headers**: `Authorization: Bearer {accessToken}` (optional for browsing)

**Query Parameters**:
- `mode` (string, optional): `eats` | `live` (default: `eats`)
- `page` (number, optional): Page number (default: 1)
- `limit` (number, optional): Items per page (default: 20, max: 100)
- `categoryId` (string, optional): Filter by category
- `healthTags` (string[], optional): Filter by health tags (for `live` mode)
  - Values: `high-protein`, `low-carb`, `keto`, `vegan`, `gluten-free`
- `latitude` (number, optional): User's latitude
- `longitude` (number, optional): User's longitude
- `radius` (number, optional): Search radius in km (default: 5)
- `search` (string, optional): Search term (powered by MongoDB text search)

**Implementation Logic**:
```javascript
// Consumer API Controller
async getRestaurants(req, res) {
  const { mode, page, limit, healthTags, latitude, longitude } = req.query;
  
  // 1. Fetch restaurants from POS V2 API
  const posResponse = await axios.get(`${POS_V2_URL}/api/stores`, {
    params: { page, limit }
  });
  
  // 2. Get additional metadata from Consumer DB
  const restaurantIds = posResponse.data.map(r => r._id);
  const metadata = await RestaurantMetadata.find({
    restaurantId: { $in: restaurantIds }
  });
  
  // 3. Merge data and apply mode filter
  let restaurants = posResponse.data.map(restaurant => {
    const meta = metadata.find(m => m.restaurantId === restaurant._id);
    return {
      ...restaurant,
      healthTags: meta?.healthTags || [],
      videoHeader: meta?.videoHeader,
      vibeScore: meta?.vibeScore || 0,
      ratings: meta?.ratings || { average: 0, count: 0 }
    };
  });
  
  // 4. Filter by mode
  if (mode === 'live' && healthTags?.length) {
    restaurants = restaurants.filter(r => 
      r.healthTags.some(tag => healthTags.includes(tag))
    );
  }
  
  // 5. Calculate distance if coordinates provided
  if (latitude && longitude) {
    restaurants = restaurants.map(r => ({
      ...r,
      distanceKm: calculateDistance(latitude, longitude, r.latitude, r.longitude)
    }));
  }
  
  return res.json({ data: restaurants });
}
```

**Configuration Required**:
- Environment variable: `POS_V2_API_URL=http://localhost:8080` (dev)
- Consumer DB collection: `RestaurantMetadata` (stores health tags, videos, ratings)

3. Phapay SDK - Payment processing

**🔗 External Endpoints**: 
- POS V2 API → `POST /api/orders/create`
- Phapay SDK → `paymentGateway.createSession()`

**Description**: Converts cart to order and initiates payment

**Headers**: `Authorization: Bearer {accessToken}`

**Request Body**:
```json
{
  "paymentMethod": "phapay",
  "tipAmount": 10000,
  "loyaltyPointsToRedeem": 100,
  "customerInfo": {
    "name": "Somsack",
    "phone": "8562093352677"
  }
}
```

**Response** (200 OK):
```json
{
  "orderId": "order_001",
  "paymentUrl": "https://payment.phapay.com/checkout/xyz",
  "qrCode": "data:image/png;base64,...",
  "totalAmount": 58000,
```

**Implementation Logic**:
```javascript
// Consumer API Controller
async checkoutCart(req, res) {
  const { cartId } = req.params;
  const { paymentMethod, tipAmount, loyaltyPointsToRedeem } = req.body;
  const userId = req.user.id;
  
  // 1. Get cart from Consumer DB
  const cart = await Cart.findById(cartId).populate('items');
  if (!cart || cart.userId !== userId) {
    return res.status(404).json({ error: 'Cart not found' });
  }
  
  // 2. Calculate totals
  let subtotal = cart.items.reduce((sum, item) => sum + item.itemTotal, 0);
  let discount = cart.discount || 0;
  let loyaltyDiscount = loyaltyPointsToRedeem * 50; // 10 pts = 500 LAK
  let total = subtotal - discount - loyaltyDiscount + (tipAmount || 0);
  
  // 3. Create order in Consumer DB
  const order = await Order.create({
    userId,
    orderCode: generateOrderCode(),
    orderType: cart.orderType,
    productType: 'eats',
    restaurantId: cart.restaurantId,
    tableId: cart.tableId,
    items: cart.items,
    subtotal,
    discount,
    tip: tipAmount,
    total,
    pointsRedeemed: loyaltyPointsToRedeem,
    paymentMethod,
    paymentStatus: 'pending',
    status: 'pending',
    posSyncStatus: 'pending'
  });
  
  // 4. Deduct loyalty points (if used)
  if (loyaltyPointsToRedeem > 0) {
    await LoyaltyTransaction.create({
      userId,
      type: 'redeem',
      amount: -loyaltyPointsToRedeem,
      sourceId: order._id,
      description: `Redeemed for ${loyaltyDiscount} LAK discount`
    });
  }
  
  // 5. Send order to POS V2 API
  try {
    const posOrder = await axios.post(`${POS_V2_URL}/api/orders/create`, {
      restaurantId: cart.restaurantId,
      tableId: cart.tableId,
      items: cart.items.map(item => ({
        menuItemId: item.menuItemId,
        quantity: item.quantity,
        modifiers: item.modifiers,
        specialInstructions: item.specialInstructions
      })),
      orderSource: 'consumer_app',
      externalOrderId: order._id.toString()
    });
    
    // Update order with POS order ID
    order.posOrderId = posOrder.data.orderId;
    order.posSyncStatus = 'synced';
    await order.save();
    
  } catch (error) {
    // Log error but don't fail checkout
    console.error('POS sync failed:', error);
    order.posSyncStatus = 'failed';
    await order.save();
  }
  
  // 6. Initiate payment with Phapay
  const paymentSession = await phapaySDK.createSession({
    orderId: order._id.toString(),
    amount: total,
    returnUrl: 'appzap://payment/callback',
    webhookUrl: `${CONSUMER_API_URL}/v1/payments/webhook/phapay`
  });
  
  // 7. Delete cart
  await Cart.findByIdAndDelete(cartId);
  
  // 8. Return response
  return res.json({
    orderId: order._id,
    paymentUrl: paymentSession.url,
    qrCode: paymentSession.qrCode,
    totalAmount: total,
    breakdown: {
      subtotal,
      discount,
      tip: tipAmount,
      loyaltyDiscount
    },
    estimatedPrepTime: 15,
    expiresAt: new Date(Date.now() + 15 * 60000) // 15 min
  });
}
```

**Database Operations**:
- **READ**: `Carts` collection
- **WRITE**: Create record in `Orders` collection
- **WRITE**: Create record in `LoyaltyTransactions` (if points redeemed)
- **DELETE**: Delete cart after checkout

**External API Calls**:
1. **POS V2 API**: Send order to kitchen (async, non-blocking)
2. **Phapay**: Create payment session

**Error Handling**:
- If POS sync fails: Continue checkout, mark `posSyncStatus: 'failed'`, retry later
- If payment init fails: Roll back order creation, return error

```javascript
  // 1. Get user's active profile from Consumer DB
  const user = await User.findById(userId);
  const activeProfile = user.activeProfile; // 'personal' or 'merchant'
  
  // 2. Authenticate with Supplier API (cache token)
  let supplierToken = await redis.get(`supplier:token:${user.phone}`);
  if (!supplierToken) {
    const authResponse = await axios.post(`${SUPPLIER_API_URL}/user/authenticate/token`, {
      phone: user.phone,
      exchangeKey: process.env.SUPPLIER_EXCHANGE_KEY
    });
    supplierToken = authResponse.data.accessToken;
    await redis.setex(`supplier:token:${user.phone}`, 86400, supplierToken); // 24h cache
  }
  
  // 3. Fetch products from Supplier API
  const supplierResponse = await axios.get(
    `${SUPPLIER_API_URL}/mobile/product/get/skip/${skip}/limit/${limit}`,
    {
      headers: { Authorization: `Bearer ${supplierToken}` },
      params: {
        categoryId,
        createdBy: '6683700875395b0f0741b48d' // Hardcoded seller ID
      }
    }
  );
  
  // 4. Transform products based on profile
  const products = supplierResponse.data.data.map(product => {
    // Filter variants by profile type
    let variants = product.priceRecords;
    
    if (activeProfile === 'personal') {
      // B2C: Show retail variants (small packs)
      variants = variants.filter(v => 
        ['ກິໂລ', '500g', '1kg'].includes(v.unitName)
      ).map(v => ({
        id: v._id,
        size: v.unitName,
        unit: v.unitName,
        price: v.buyPrice, // Retail price
        priceType: 'retail',
        stock: 50, // Mock stock for now
        isAvailable: true
      }));
    } else {
      // B2B: Show wholesale variants (bulk packs)
      variants = variants.map(v => ({
        id: v._id,
        size: '10kg', // Transform to bulk size
        unit: v.unitName,
        price: v.buyPrice * 9.5, // Wholesale discount (5% off bulk)
        pricePerUnit: v.buyPrice * 0.95,
        priceType: 'wholesale',
        minimumOrder: 10,
        stock: 500,
        isAvailable: true,
        creditTermsAvailable: true
      }));
    }
    
    return {
      id: product._id,
      name: product.productName,
      nameEn: product.productName,
      description: product.description,
      images: product.images,
      category: {
        id: product.categoryId._id,
        name: product.categoryId.categoryName,
        nameEn: product.categoryId.categoryName
      },
      variants,
      tags: ['fresh', 'local'],
      promotions: product.promotionDetail || []
    };
  });
  
  // 5. Return response with view mode
  return res.json({
    viewMode: activeProfile === 'personal' ? 'retail' : 'wholesale',
    products,
    pagination: {
      skip,
      limit,
      total: supplierResponse.data.total
    }
  });
}
```

```javascript
// Consumer API Controller
async checkoutMarketCart(req, res) {
  const { cartId } = req.params;
  const { deliveryAddress, paymentMethod, creditTermDays } = req.body;
  const userId = req.user.id;
  
  // 1. Get cart and user from Consumer DB
  const cart = await MarketCart.findById(cartId).populate('items');
  const user = await User.findById(userId);
  
  if (!cart || cart.userId !== userId) {
    return res.status(404).json({ error: 'Cart not found' });
  }
  
  // 2. Create order in Consumer DB
  const order = await MarketOrder.create({
    userId,
    orderCode: generateMarketOrderCode(),
    profileType: cart.profileType, // 'personal' or 'merchant'
    restaurantId: cart.restaurantId, // if merchant
    items: cart.items,
    subtotal: cart.subtotal,
    deliveryFee: cart.deliveryFee,
    total: cart.total,
    deliveryAddress,
    paymentMethod,
    creditTermDays: paymentMethod === 'credit_term' ? creditTermDays : null,
    paymentStatus: paymentMethod === 'credit_term' ? 'pending_invoice' : 'pending',
    status: 'pending',
    supplierSyncStatus: 'pending'
  });
  
  // 3. Determine which Supplier account to use based on profile
  let supplierCustomerId;
  let customerInfo;
  let tokenCacheKey;
  
  if (cart.profileType === 'personal') {
    // B2C: Use personal Supplier account
    supplierCustomerId = user.supplierId;
    
    // If no personal Supplier account, create one
    if (!supplierCustomerId) {
      const supplierAuth = await supplierApi.authenticateForPersonalUser(userId);
      supplierCustomerId = supplierAuth.customerId;
      user.supplierId = supplierCustomerId;
      await user.save();
    }
    
    customerInfo = {
      name: user.fullName,
      phone: normalizePhoneForSupplier(user.phone),
      village: deliveryAddress.village,
      district: deliveryAddress.district,
      province: deliveryAddress.province,
      latitude: deliveryAddress.latitude?.toString(),
      longitude: deliveryAddress.longitude?.toString()
    };
    
    tokenCacheKey = `supplier:token:personal:${userId}`;
    
  } else {
    // B2B: Use restaurant's Supplier account
    const merchantProfile = user.merchantProfiles.find(
      p => p.restaurantId === cart.restaurantId
    );
    
    if (!merchantProfile) {
      return res.status(403).json({ 
        error: 'No merchant profile found for this restaurant' 
      });
    }
    
    supplierCustomerId = merchantProfile.supplierCustomerId;
    
    // If restaurant not linked to Supplier, link it now
    if (!supplierCustomerId) {
      const supplierAuth = await supplierApi.authenticateForRestaurant(cart.restaurantId);
      supplierCustomerId = supplierAuth.customerId;
      merchantProfile.supplierCustomerId = supplierCustomerId;
      await user.save();
    }
    
    // Get restaurant details from POS V2
    const restaurant = await posV2Api.getRestaurant(cart.restaurantId);
    
    customerInfo = {
      name: restaurant.name,
      phone: normalizePhoneForSupplier(restaurant.phone),
      village: deliveryAddress.village || restaurant.address.village,
      district: deliveryAddress.district || restaurant.address.district,
      province: deliveryAddress.province || restaurant.address.province,
      latitude: deliveryAddress.latitude?.toString(),
      longitude: deliveryAddress.longitude?.toString()
    };
    
    tokenCacheKey = `supplier:token:restaurant:${cart.restaurantId}`;
  }
  
  // 4. Get Supplier API token
  let supplierToken = await redis.get(tokenCacheKey);
  if (!supplierToken) {
    const authResponse = await axios.post(`${SUPPLIER_API_URL}/user/authenticate/token`, {
      phone: customerInfo.phone,
      exchangeKey: process.env.SUPPLIER_EXCHANGE_KEY
    });
    supplierToken = authResponse.data.accessToken;
    await redis.setex(tokenCacheKey, 86400, supplierToken);
  }
  
  // 5. Submit order to Supplier API
  try {
    const supplierOrderPayload = {
      items: cart.items.map(item => ({
        productId: item.productId,
        name: item.name,
        quantity: item.quantity,
        unit: item.unit,
        price: item.price,
        priceRecordId: item.variantId,
        note: item.note
      })),
      customerId: supplierCustomerId, // Personal or Restaurant Supplier customer ID
      sellerId: '6683700875395b0f0741b48d', // Hardcoded seller ID
      customerInfo
    };
    
    const supplierResponse = await axios.post(
      `${SUPPLIER_API_URL}/order/lll-create-order`,
      supplierOrderPayload,
      {
        headers: { Authorization: `Bearer ${supplierToken}` }
      }
    );
    
    // Update order with Supplier order ID
    order.supplierOrderId = supplierResponse.data.data.id;
    order.supplierSyncStatus = 'synced';
    await order.save();
    
  } catch (error) {
    console.error('Supplier sync failed:', error);
    order.supplierSyncStatus = 'failed';
    await order.save();
    // Don't fail checkout, retry later
  }
  
  // 6. Handle payment
  let paymentUrl = null;
  
  if (paymentMethod === 'phapay') {
    // Initiate payment (both B2C and B2B can use Phapay)
    const paymentSession = await phapaySDK.createSession({
      orderId: order._id.toString(),
      amount: order.total,
      returnUrl: 'appzap://payment/callback',
      webhookUrl: `${CONSUMER_API_URL}/v1/payments/webhook/phapay`
    });
    paymentUrl = paymentSession.url;
    
  } else if (paymentMethod === 'credit_term') {
    // B2B: Generate invoice (if required)
    if (req.body.requireTaxInvoice) {
      const invoiceUrl = await generateTaxInvoice(order, req.body.taxInfo);
      order.invoiceUrl = invoiceUrl;
      await order.save();
    }
  }
  
  // 7. Delete cart
  await MarketCart.findByIdAndDelete(cartId);
  
  // 8. Send push notification
  await sendPushNotification(userId, {
    title: 'Order Placed',
    body: `Your market order ${order.orderCode} is being processed`
  });
  
  // 9. Return response
  return res.json({
    orderId: order._id,
    orderCode: order.orderCode,
    status: order.status,
    paymentMethod,
    paymentUrl,
    totalAmount: order.total,
    estimatedDelivery: '2025-12-19 08:00-12:00',
    trackingUrl: `${CONSUMER_API_URL}/v1/market/orders/${order._id}`,
    profileType: cart.profileType,
    supplierLinked: true
  });
}
```

**Database Operations**:
- **READ**: `MarketCarts` collection, `Users` collection
- **WRITE**: Create record in `MarketOrders` collection
- **WRITE**: Update `user.supplierId` (if personal) or `merchantProfile.supplierCustomerId` (if merchant)
- **DELETE**: Delete cart after checkout

**External API Calls**:
1. **POS V2 API**: Get restaurant details (B2B only)
2. **Supplier API**: Authenticate (cached), submit order
3. **Phapay**: Create payment session (optional for B2B)

**Key Differences by Profile**:

| Aspect | Personal (B2C) | Merchant (B2B) |
|--------|---------------|----------------|
| **Supplier Account** | `user.supplierId` | `merchantProfile.supplierCustomerId` |
| **Customer Name** | User's personal name | Restaurant name |
| **Customer Phone** | User's personal phone | Restaurant phone |
| **Delivery Address** | Personal address | Restaurant address (or custom) |
| **Payment** | Immediate via Phapay | Phapay or Credit Terms |
| **Invoice** | Optional | Business tax invoice available |
| **POS V2 Call** | ❌ Not needed | ✅ Get restaurant details |

**Supplier Account Auto-Creation**:
- **Personal**: If `user.supplierId` is null, automatically create using personal phone
- **Merchant**: If `merchantProfile.supplierCustomerId` is null, automatically create using restaurant phone

## Error Handling

### HTTP Status Codes

| Code | Meaning | Usage |
|------|---------|-------|
| 200 | OK | Successful GET, PUT, DELETE |
| 201 | Created | Successful POST |
| 400 | Bad Request | Invalid request body/params |
| 401 | Unauthorized | Missing/invalid token |
| 403 | Forbidden | Valid token but insufficient permissions |
| 404 | Not Found | Resource doesn't exist |
| 409 | Conflict | Duplicate resource (e.g., phone already exists) |
| 422 | Unprocessable Entity | Validation errors |
| 429 | Too Many Requests | Rate limit exceeded |
| 500 | Internal Server Error | Server error |
| 502 | Bad Gateway | Downstream API error (POS, Supplier) |
| 503 | Service Unavailable | Maintenance mode |

---

### Error Response Format

```json
{
  "error": {
    "code": "ERROR_CODE",
    "message": "Human-readable error message",
    "details": {
      "field": "phone",
      "reason": "Phone number already exists"
    },
    "statusCode": 400
  }
}
```

### Security Considerations

1. **Authentication:**
   - JWT tokens with 24h expiry (access), 30d expiry (refresh)
   - Refresh token rotation on each use
   - Rate limiting on OTP requests (3 per 5 min)

2. **API Security:**
   - Helmet.js for security headers
   - CORS whitelist for mobile app origins
   - Input validation with Joi/Yup
   - SQL injection prevention (use Mongoose)

3. **Payment Security:**
   - Webhook signature verification (Phapay)
   - HTTPS-only communication
   - PCI-DSS compliance (no card storage)

4. **Data Privacy:**
   - Phone number encryption at rest
   - PII masking in logs
   - GDPR/PDPA compliance

---

### Performance Optimization

1. **Caching Strategy:**
   - Redis cache for:
     - Restaurant listings (5 min TTL)
     - Product catalog (10 min TTL)
     - User sessions (24h TTL)
     - Supplier API tokens (7d TTL)

2. **Database Indexing:**
   - User: `phone` (unique)
   - Order: `userId`, `status`, `createdAt`
   - Product: `categoryId`, `name` (text index)

3. **API Response Optimization:**
   - Pagination for all list endpoints
   - Field filtering support (`?fields=id,name,price`)
   - Compression (gzip)

4. **Rate Limiting:**
   - Global: 100 req/min per IP
   - Auth endpoints: 3 OTP req/5min per phone
   - Payment endpoints: 10 req/min per user

6. Consumer API forwards order to Supplier API
7. If B2C: Initiate payment | If B2B: Generate invoice with credit terms

### Flow 4: Payment Webhook (Order Completion)

```
┌──────────┐   ┌──────────────┐   ┌──────────────┐   ┌──────────┐
│ Phapay   │   │  Consumer    │   │   POS V2     │   │ Consumer │
│  Server  │   │     API      │   │     API      │   │    DB    │
└─────┬────┘   └──────┬───────┘   └──────┬───────┘   └────┬─────┘
      │               │                   │                │
      │ POST /v1/     │                   │                │
      │ payments/     │                   │                │
      │  webhook      │                   │                │
      ├──────────────>│                   │                │
      │               │ Verify signature  │                │
      │               │                   │                │
      │               │ Update Order: paymentStatus='paid' │
      │               ├───────────────────────────────────>│
      │               │                   │                │
      │               │ Award loyalty pts │                │
      │               ├───────────────────────────────────>│
      │               │                   │                │
      │ [If Eats]     │                   │                │
      │               │ PUT /api/orders/  │                │
      │               │  {id}/status      │                │
      │               ├──────────────────>│                │
      │               │<──────────────────┤                │
      │               │ Print to kitchen  │                │
      │               │                   │                │
      │ [If Market]   │                   │                │
      │               │ (Supplier API already notified)    │
      │               │                   │                │
      │<──────────────┤                   │                │
      │  200 OK       │                   │                │
      │               │                   │                │
```

**Data Flow:**
1. Phapay sends webhook to Consumer API
2. Consumer API updates order status in Consumer DB
3. Consumer API awards loyalty points
4. If Eats order: Consumer API notifies POS V2 to print to kitchen
5. If Market order: Supplier API already has order, no additional action
